## 開發注意事項

### 1. 程式碼規範
- 使用駝峰式命名
- 保持函數單一職責
- 添加適當的註解
- 避免全域變數

### 2. 資料結構
寵物資料格式：
```javascript
{
    name: String,          // 寵物名稱
    traits: {             // 特徵
        personality: Array,    // 個性
        lifestyle: Array,     // 生活型態
        space: Array          // 空間需求
    },
    image: String,        // 圖片URL
    description: String   // 描述文字
}
```

### 3. 效能考量
- 最小化 DOM 操作
- 優化圖片載入
- 使用事件委派
- 避免不必要的重複計算

### 4. 測試重點
- 跨瀏覽器相容性
- 移動裝置響應性
- 表單驗證
- 錯誤處理
